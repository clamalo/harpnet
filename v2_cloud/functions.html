Function Documentation for /v2/src/

This document provides detailed information about the functions in the /v2/src/ subdirectory. Each function is named after the file it resides in. The documentation includes the purpose of each function, its inputs and outputs, and their respective types.

create_grid_tiles()

Purpose

Generates a dictionary of grid tiles covering a specified geographical area. Each tile represents a subdomain defined by latitude and longitude boundaries.

Inputs

  • None

Outputs

  • grid_domains (dict): A dictionary where each key is a tile index (int), and the value is a list of coordinates [min_lat, max_lat, min_lon, max_lon] defining the tile boundaries.

generate_dataloaders()

Purpose

Generates PyTorch DataLoaders for training and testing datasets based on preprocessed NumPy arrays for a specified tile and date range.

Inputs

  • tile (int): The tile index for which to generate data loaders.
  • first_month (tuple): A tuple (year, month) representing the starting month (e.g., (1979, 1)).
  • last_month (tuple): A tuple (year, month) representing the ending month.
  • train_test_ratio (float): Ratio (between 0 and 1) of data to be used for training; the remaining data is used for testing.

Outputs

  • train_dataloader (torch.utils.data.DataLoader): DataLoader for the training dataset.
  • test_dataloader (torch.utils.data.DataLoader): DataLoader for the testing dataset.

get_coordinates()

Purpose

Generates coarse and fine latitude and longitude coordinate arrays for a specified tile, including padded coordinates for interpolation purposes.

Inputs

  • tile (int): The tile index for which to generate coordinates.

Outputs

  • coarse_lats_pad (numpy.ndarray): 1D array of coarse latitude coordinates with padding.
  • coarse_lons_pad (numpy.ndarray): 1D array of coarse longitude coordinates with padding.
  • coarse_lats (numpy.ndarray): 1D array of coarse latitude coordinates without padding.
  • coarse_lons (numpy.ndarray): 1D array of coarse longitude coordinates without padding.
  • fine_lats (numpy.ndarray): 1D array of fine (high-resolution) latitude coordinates.
  • fine_lons (numpy.ndarray): 1D array of fine (high-resolution) longitude coordinates.

UNetWithAttention()

Purpose

Defines a U-Net architecture with attention mechanisms for performing image segmentation or super-resolution tasks. The model includes residual convolutional blocks and attention blocks to enhance feature extraction and focus on relevant regions.

Inputs

  • in_channels (int): Number of input channels (e.g., 1 for grayscale images).
  • out_channels (int): Number of output channels (e.g., 1 for single-channel output).
  • output_shape (tuple of int, optional): The desired output spatial dimensions (default is (64, 64)).

Methods

  • forward(x): Performs a forward pass through the network.
    • x (torch.Tensor): Input tensor with shape (batch_size, in_channels, H, W).
    • Returns: torch.Tensor with shape (batch_size, output_shape[0], output_shape[1]).

Inputs

Outputs

realtime_ecmwf()

Purpose

Retrieves and processes real-time ECMWF forecast data for specified date, cycle, and forecast steps. The data is processed to compute incremental total precipitation and formatted for further use.

Inputs

  • datestr (str): Date string in 'YYYYMMDD' format (e.g., '20241017').
  • cycle (str): Forecast cycle time as a string (e.g., '00', '12').
  • frames (list of int): List of forecast lead times (in hours) to retrieve (e.g., [3, 6, 9]).
  • ingest (bool): If True, retrieves data from the ECMWF server; if False, uses existing data.

Outputs

  • ds (xarray.Dataset): Processed dataset containing the total precipitation (tp) and other variables.

realtime_gfs()

Purpose

Retrieves and processes real-time GFS forecast data for specified date, cycle, and forecast steps. The data is processed to compute incremental total precipitation and formatted for further use.

Inputs

  • datestr (str): Date string in 'YYYYMMDD' format (e.g., '20241017').
  • cycle (str): Forecast cycle time as a string (e.g., '00', '12').
  • frames (list of int): List of forecast lead times (in hours) to retrieve (e.g., [3, 6, 9]).
  • ingest (bool): If True, retrieves data from the GFS server; if False, uses existing data.

Outputs

  • ds (xarray.Dataset): Processed dataset containing the total precipitation (tp) and other variables.

setup()

Purpose

Prepares the necessary directory structure for a specified tile by creating directories for data storage and model checkpoints if they do not already exist.

Inputs

  • tile (int): The tile index for which to set up directories.

Outputs

  • None: The function creates directories on the file system as a side effect.

train_test()

Purpose

Trains a neural network model using the provided training and testing data loaders. After each epoch, the model's performance is evaluated, and a checkpoint containing the model state and loss metrics is saved.

Inputs

  • tile (int): The tile index, used for saving model checkpoints specific to the tile.
  • train_dataloader (torch.utils.data.DataLoader): DataLoader for the training dataset.
  • test_dataloader (torch.utils.data.DataLoader): DataLoader for the testing dataset.
  • epochs (int, optional): Number of training epochs (default is 20).

Outputs

  • None: The function trains the model and saves checkpoints to the file system.

xr_to_np()

Purpose

Converts Xarray datasets to NumPy arrays for a specified tile and date range. The function interpolates the datasets to both coarse and fine grids and saves the resulting arrays along with the timestamps to disk.

Inputs

  • tile (int): The tile index for which to process data.
  • start_month (tuple): A tuple (year, month) representing the starting month (e.g., (1979, 10)).
  • end_month (tuple): A tuple (year, month) representing the ending month.

Outputs

  • None: The function saves NumPy arrays to disk at specified file paths.
